# 校园跑腿系统 — 代码功能说明

## 项目概述

基于 Python tkinter 的桌面应用程序，使用 SQL Server 数据库存储数据，实现校园内的跑腿订单发布、接单配送、评价管理等核心功能。系统支持三种角色：**需求方**（发布任务）、**配送员**（接单配送）、**管理员**（后台管理）。

## main.py 代码结构详解

### 1. 导入模块（第 1-10 行）

| 模块 | 用途 |
|------|------|
| `tkinter` / `ttk` | Python 标准 GUI 库，构建桌面界面 |
| `messagebox` | 弹出提示、警告、错误对话框 |
| `datetime` | 处理订单创建、完成等时间戳 |
| `uuid` | 生成唯一订单 ID（取前 8 位） |
| `pyodbc` | 连接 SQL Server 数据库 |

### 2. 数据库配置与连接（第 12-28 行）

- **`DB_CONN_STR`**：SQL Server 连接字符串，指定本地 SQLEXPRESS 实例、数据库 `CampusErrand`，使用 Windows 身份验证。
- **`get_conn()`**：创建并返回一个新的数据库连接。
- **`dict_from_row(row, columns)`**：工具函数，将数据库查询结果行（元组）与列名列表合并为字典，方便按字段名访问数据。

### 3. 数据层 — `Database` 类（第 32-348 行）

负责所有数据库读写操作，每个方法内独立创建和关闭连接。

#### 用户相关方法

| 方法 | 功能 |
|------|------|
| `register(username, password, role, name, phone, student_id)` | 注册新用户，先检查用户名是否已存在 |
| `login(username, password)` | 用户登录验证，返回用户信息（不含密码） |
| `get_user(username)` | 获取单个用户详细信息 |
| `get_all_users()` | 获取所有用户列表，按注册时间倒序 |

#### 订单相关方法

| 方法 | 功能 |
|------|------|
| `create_order(...)` | 创建新订单，生成 UUID 前 8 位作为订单号，状态默认为"待接单" |
| `get_order(order_id)` | 根据订单 ID 查询订单详情 |
| `get_available_orders()` | 获取所有"待接单"状态的订单，按赏金和发布时间排序 |
| `get_orders_by_user(username)` | 获取某个用户发布的所有订单 |
| `get_orders_by_delivery(username)` | 获取某个配送员接的所有订单 |
| `get_all_orders()` | 管理员使用，获取全部订单 |
| `accept_order(order_id, delivery_person)` | 配送员接单，将订单状态改为"配送中"，记录配送员和接单时间 |
| `complete_order(order_id)` | 配送员确认送达，状态改为"已完成"，配送员完成单数 +1 |
| `cancel_order(order_id)` | 需求方取消订单（仅"待接单"状态可取消） |

#### 评价相关方法

| 方法 | 功能 |
|------|------|
| `add_review(order_id, rating, comment)` | 添加评价，同时更新配送员的平均评分 |
| `review_exists(order_id)` | 检查某订单是否已被评价 |
| `get_reviews_by_requester(username)` | 获取某个用户给出的所有评价 |

### 4. 全局变量（第 348-349 行）

- **`db = Database()`**：全局数据库实例
- **`current_user`**：存储当前登录用户信息，初始为 `None`

### 5. 页面框架 — `App` 类（第 354-394 行）

继承 `tk.Tk`，是应用程序的主窗口。

| 特性 | 说明 |
|------|------|
| 窗口标题 | "校园跑腿系统" |
| 初始尺寸 | 900×650 像素，最小 800×600 |
| 页面管理 | 通过 `show_page(page_name)` 切换页面，页面实例缓存在 `self.pages` 字典中 |
| 页面刷新 | 切换时自动调用页面的 `on_show()` 方法刷新数据 |
| 登出清理 | `clear_pages()` 清除所有缓存的页面实例 |

### 6. 公共组件（第 399-432 行）

#### HeaderFrame — 顶部导航栏

- 蓝色背景（`#4A90D9`），显示页面标题
- 可选"返回"按钮
- 右上角显示当前用户名和角色

#### InfoCard — 信息卡片

- 白色背景卡片，显示标签 + 数值（用于统计数据展示）

### 7. 登录与注册页面（第 437-598 行）

#### LoginPage — 登录页

- 居中布局，带 Logo 标语 "便捷校园 · 轻松跑腿"
- 用户名/密码输入框（默认填充 admin/123456 方便测试）
- 回车键可直接触发登录
- 登录成功后根据角色跳转到不同首页
- 底部提供快速体验账号提示

#### RegisterPage — 注册页

- 表单字段：用户名、密码、真实姓名、手机号、学号（可选）
- 角色选择：需求方 / 配送员（单选按钮）
- 注册成功后跳转回登录页

### 8. 需求方页面（第 603-880 行）

#### RequesterHomePage — 需求方首页

- 顶栏功能按钮：发布订单、我的订单、评价配送、个人中心
- 下方展示最近的 10 条订单（带状态标签和操作按钮）
- 可对"待接单"订单取消，对"已完成"订单评价

#### NewOrderPage — 发布订单页

- 支持滚动的内容区域（Canvas + Scrollbar）
- 表单：订单类型（快递代取/外卖代送/文件代送/其他）、详细描述、取件地点、送达地点、跑腿费、备注
- 跑腿费进行数字有效性校验

#### RequesterOrderListPage — 我的订单列表

- 全部订单列表，支持按状态筛选（全部/待接单/配送中/已完成/已取消）
- 配送中的订单显示配送员姓名
- 已完成的订单可点击评价

### 9. 配送员页面（第 884-1148 行）

#### DeliveryHomePage — 配送员首页

- 顶部显示配送员信息卡片：姓名、评分、已完成单数
- 功能按钮：浏览订单、我的配送、我的收入
- 下方展示待接订单列表，可直接点击"接单"

#### DeliveryBrowsePage — 浏览订单页

- 展示所有待接订单，信息更完整（含跑腿费、发布时间、备注）
- 点击"接单"后跳回配送员首页

#### DeliveryOrderListPage — 我的配送列表

- 展示该配送员接过的所有订单
- 配送中的订单可点击"确认送达"

#### DeliveryIncomePage — 我的收入页

- 统计卡片：已完成订单数、总收入、评分
- 下方列出已完成订单明细（含完成时间）

### 10. 评价页面（第 1153-1239 行）

#### ReviewPage — 写评价页

- 1-5 星评分选择（默认 5 星）
- 评价内容输入框（多行文本）
- 提交后自动更新配送员评分

#### ReviewListPage — 评价列表页

- 展示当前用户发出的所有评价
- 显示星级、评语、对应订单信息

### 11. 个人中心 — `ProfilePage`（第 1244-1293 行）

- 展示当前用户的详细信息（用户名、姓名、手机号、学号、角色）
- 配送员额外显示评分和已完成单数
- "退出登录"按钮（确认后清除页面缓存，返回登录页）

### 12. 管理员页面 — `AdminPage`（第 1298-1441 行）

- 顶部统计卡片：总用户数、总订单数、进行中订单数、已完成订单数
- 双 Tab 切换：

  **订单管理** — 表格显示全部订单（订单号、类型、需求方、配送员、状态、金额、时间）

  **用户管理** — 表格显示全部用户（用户名、姓名、角色、手机号、学号）

### 13. 启动入口（第 1447-1450 行）

```python
if __name__ == "__main__":
    app = App()
    app.mainloop()
```

创建 `App` 实例并启动 tkinter 主事件循环。

## 数据表结构（SQL Server）

程序依赖以下两张表（需在 `CampusErrand` 数据库中预先创建）：

### Users 表

| 字段 | 类型 | 说明 |
|------|------|------|
| username | VARCHAR(50) PK | 用户名 |
| password | VARCHAR(100) | 密码 |
| role | VARCHAR(20) | 角色：requester / delivery / admin |
| name | NVARCHAR(50) | 真实姓名 |
| phone | VARCHAR(20) | 手机号 |
| student_id | VARCHAR(50) | 学号 |
| rating | FLOAT | 配送员平均评分 |
| completed | INT | 配送员完成单数 |
| balance | DECIMAL | 余额 |
| created_at | DATETIME | 注册时间 |

### Orders 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) PK | 订单号（UUID 前 8 位） |
| requester | VARCHAR(50) | 需求方用户名 |
| type | NVARCHAR(50) | 订单类型 |
| details | NVARCHAR(500) | 详细描述 |
| pickup | NVARCHAR(200) | 取件地点 |
| dropoff | NVARCHAR(200) | 送达地点 |
| reward | DECIMAL | 跑腿费 |
| note | NVARCHAR(200) | 备注 |
| status | NVARCHAR(20) | 状态：待接单/配送中/已完成/已取消 |
| delivery_person | VARCHAR(50) | 配送员用户名 |
| created_at | VARCHAR(20) | 创建时间 |
| accepted_at | VARCHAR(20) | 接单时间 |
| completed_at | VARCHAR(20) | 完成时间 |

### Reviews 表

| 字段 | 类型 | 说明 |
|------|------|------|
| order_id | VARCHAR(50) | 订单号 |
| requester | VARCHAR(50) | 评价人 |
| delivery_person | VARCHAR(50) | 被评价配送员 |
| rating | INT | 评分（1-5） |
| comment | NVARCHAR(500) | 评语 |

## 核心业务流程

```
需求方发布订单 → 订单状态: 待接单
                        ↓
配送员浏览并接单 → 订单状态: 配送中
                        ↓
配送员确认送达 → 订单状态: 已完成
                        ↓
需求方评价配送员 → 更新配送员评分
```
