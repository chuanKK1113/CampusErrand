# 校园跑腿系统 - 使用说明

涵盖微信小程序端和 Web 管理后台端。

---

## 一、小程序端

### 1. 登录与注册

| 测试账号 | 密码 | 角色 |
|---------|------|------|
| `zhangsan` | `123456` | 需求方（发布任务） |
| `lisi` | `123456` | 配送员（接单跑腿） |

---

### 2. 底部导航栏

底部三个 Tab：
| 图标 | 名称 | 说明 |
|------|------|------|
| 🏠 | 首页 | 角色自适应首页 |
| 📋 | 订单 | 角色自适应订单列表 |
| 👤 | 我的 | 个人中心 |

---

### 3. 需求方功能（zhangsan）

#### 首页
- 顶部显示 **欢迎信息** 和 **角色标识**
- **快捷操作：**
  - 📝 **发布订单** — 创建新的跑腿任务
  - 📋 **我的订单** — 查看所有订单
  - ⭐ **我的评价** — 查看已做出的评价
- **最近订单：** 显示最近 5 条订单记录，点击进入详情

#### 发布订单
1. 填写以下信息：
   - **订单类型：** 快递代取 / 外卖代送 / 文件代送 / 其他
   - **详细描述：** 跑腿具体内容
   - **取件地点：** 取件位置
   - **送达地点：** 送达位置
   - **配送费：** 报酬（须大于 0）
   - **备注：** 其他说明（选填）
2. 点击「发布订单」提交
3. 发布成功后自动返回

#### 订单列表
- 顶部 **筛选标签：** 全部 / 待接单 / 配送中 / 已完成 / 已取消
- 每个订单卡片显示：类型、状态（彩色标签）、描述、取送地点、配送费、时间
- 点击订单进入 **详情页**

#### 订单详情
- **状态卡片：** 顶部颜色标识
  - 🟡 待接单 → 可取消
  - 🔵 配送中 → 显示配送员信息
  - 🟢 已完成 → 可评价配送员
  - ⚪ 已取消
- **订单信息：** 编号、类型、描述、取件地、送达地、配送费
- **配送信息：** 配送员、接单时间、完成时间
- **操作按钮：** 根据状态动态显示

#### 评价配送员
1. 进入已完成订单详情
2. 点击「评价配送员」
3. 点击星星评分（1-5 星）
4. 填写评价内容（选填）
5. 提交评价（不可修改）

#### 个人中心
- **账户信息：** 用户名、姓名、手机号、学号、角色
- **功能入口：** ⭐ 我的评价
- **退出登录：** 点击确认退出

---

### 4. 配送员功能（lisi）

#### 首页
- **我的数据：** 已完成单数、总收入、评分
- **快捷操作：**
  - 🔍 **浏览订单** — 查看所有待接单
  - 🚚 **我的配送** — 查看已接配送
  - 💰 **收入统计** — 收入明细
- **配送中的订单：** 当前正在配送的订单

#### 接单流程
1. 在「订单」Tab 选择「可接单」标签
2. 浏览待接订单（按配送费从高到低排序）
3. 点击订单查看详情
4. 点击「接单」确认
5. 订单进入「配送中」状态

#### 完成配送
1. 在「配送中」找到订单
2. 点击进入详情
3. 配送完成后点击「确认送达」
4. 配送费自动计入收入统计

#### 收入统计
- 已完成订单数、总收入
- 平均评分、总完成数
- 已完成订单明细列表

#### 个人中心
- **账户信息**
- **功能入口：** 💰 收入统计、⭐ 我的评价（查看用户评价）
- **退出登录**

---

### 5. 订单状态流

```
待接单 → 配送中 → 已完成
  ↓
 已取消
```

| 状态 | 颜色 | 说明 |
|------|------|------|
| 待接单 | 🟡 黄色 | 等待配送员接单 |
| 配送中 | 🔵 蓝色 | 配送员正在配送 |
| 已完成 | 🟢 绿色 | 已送达确认 |
| 已取消 | ⚪ 灰色 | 需求方取消 |

## 三、常见问题

**小程序无法连接？**
1. 确保后端已启动：`cd backend && python main.py`
2. 微信开发者工具 → 详情 → 本地设置 → 勾选「不校验合法域名」
3. 检查 `app.js` 中 `baseUrl` 是否指向 `http://127.0.0.1:8000`

**发布订单无响应？**
- 配送费必须大于 0
- 所有必填项（描述、取件地、送达地）不能为空

**配送员看不到可接单？**
- 需要有需求方已发布且状态为「待接单」的订单

**管理后台显示加载失败？**
- 确保后端正运行，`npm run dev` 已启动
- 检查浏览器控制台是否有网络错误
- 刷新页面或重新登录

**数据不对或页面没更新？**
- 小程序：点击「编译」或清除缓存重新编译
- 后台：点击页面刷新按钮

**配送员无法确认送达？**
- 只有接单的配送员本人可以操作确认
- 确认后不可撤销
