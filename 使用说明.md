# 校园跑腿系统 使用说明

## 一、简介

校园跑腿系统是一款基于 Python tkinter 的桌面应用程序，为校园内的快递代取、外卖代送、文件代送等跑腿需求提供便捷的对接平台。系统包含三种角色：**需求方**（发布任务）、**配送员**（接单跑腿）、**管理员**（系统管理）。

---

## 二、运行环境

- **操作系统**：Windows 7 及以上
- **Python 版本**：Python 3.6 及以上
- **依赖库**：tkinter（Python 内置，无需额外安装）

### 启动方式

```bash
cd "d:/OneDrive/Desktop/校园跑腿系统"
py main.py
```

或者双击 `main.py` 文件运行。

## 四、功能详解

### 4.1 登录与注册

**登录页面**
- 输入用户名和密码即可登录
- 根据账号角色自动进入对应的操作界面
- 登录失败时会提示具体原因

**注册账号**
- 点击"立即注册"进入注册页面
- 填写用户名、密码、真实姓名、手机号等信息
- 选择注册身份：**需求方**或**配送员**
- 学号为选填项

---

### 4.2 需求方功能

#### 4.2.1 首页

登录后进入需求方首页，顶部包含四个功能按钮：

- **发布订单** — 创建新的跑腿任务
- **我的订单** — 查看所有订单及状态
- **评价配送** — 查看已提交的评价记录
- **个人中心** — 查看个人信息、退出登录

#### 4.2.2 发布订单

点击"发布订单"进入下单页面：

1. **选择订单类型**：快递代取、外卖代送、文件代送、其他
2. **填写详细信息**：
   - 详细描述（如快递单号、商家名称等）
   - 取件地点
   - 送达地点
   - 跑腿费（元）
   - 备注（选填）
3. 点击"发布订单"提交

发布成功后会自动跳转到首页，您可以在"我的订单"中查看订单状态。

#### 4.2.3 我的订单

查看所有已发布的订单，支持按状态筛选：

- **全部** — 显示所有订单
- **待接单** — 尚未被配送员接单
- **配送中** — 已被配送员接单，正在配送
- **已完成** — 已送达并确认
- **已取消** — 已取消的订单

**可执行操作：**
- **待接单**状态 → 可取消订单
- **配送中**状态 → 可查看配送员姓名
- **已完成**状态 → 可对配送员进行评价

#### 4.2.4 评价配送员

订单完成后，需求方可为配送员打分：

1. 选择星级（1-5星）
2. 填写评价内容（选填）
3. 点击"提交评价"

#### 4.2.5 个人中心

展示当前登录用户的个人信息，包括用户名、姓名、手机号、学号、角色等。点击"退出登录"可返回登录页面。

---

### 4.3 配送员功能

#### 4.3.1 首页

配送员首页顶部显示个人信息（姓名、评分、已完成单数），包含三个功能按钮：

- **浏览订单** — 查看所有待接订单
- **我的配送** — 查看已接单的配送任务
- **我的收入** — 查看收入统计和明细

首页下方展示**待接订单列表**，按跑腿费从高到低排序。

#### 4.3.2 浏览与接单

在"浏览订单"页面中：

1. 查看所有待接订单的详细信息（类型、描述、取送地点、跑腿费、备注等）
2. 点击"接单"按钮确认接单
3. 接单成功后订单状态变为"配送中"

#### 4.3.3 我的配送

查看所有已接单的配送任务，分为"配送中"和"已完成"状态。

- **配送中**的订单可点击"确认送达"完成配送
- 确认送达后订单状态变为"已完成"

#### 4.3.4 我的收入

展示收入统计信息：

- 已完成订单数
- 总收入（所有已完成订单的跑腿费总和）
- 当前评分

页面下方列出所有已完成订单的明细。

---

### 4.4 管理员功能

#### 4.4.1 首页

管理员登录后进入系统管理后台，顶部显示统计概览：

- 总用户数
- 总订单数
- 进行中的订单数
- 已完成的订单数

#### 4.4.2 订单管理

以表格形式展示系统所有订单：

- 订单号、类型、需求方、配送员、状态、金额、时间
- 支持按状态筛选查看
- 不同状态的订单用不同颜色标识

#### 4.4.3 用户管理

以表格形式展示系统所有用户：

- 用户名、姓名、角色、手机号、学号
- 角色用中文标识（需求方/配送员/管理员）

## 五、完整业务流程演示

### 场景：快递代取

以下是一个完整的业务流程示例：

**第一步：需求方发布订单**
1. 使用 `zhangsan / 123456` 登录（需求方角色）
2. 点击"发布订单"
3. 选择类型"快递代取"
4. 填写描述、取件地点（如"菜鸟驿站"）、送达地点（如"3号宿舍楼"）
5. 填写跑腿费（如 3 元）
6. 点击"发布订单"

**第二步：配送员接单**
1. 使用 `lisi / 123456` 登录（配送员角色）
2. 在首页待接订单列表中找到订单
3. 点击"接单"
4. 前往取件地点取件配送

**第三步：配送员确认送达**
1. 配送员在"我的配送"中找到该订单
2. 点击"确认送达"

**第四步：需求方评价**
1. 需求方在"我的订单"中找到已完成订单
2. 点击"评价"
3. 选择星级并填写评价
4. 提交评价

**第五步：管理员查看数据**
1. 使用 `admin / 123456` 登录
2. 查看订单管理中的全部订单记录
3. 查看用户管理中的用户列表

---

## 六、注意事项

1. 当前版本数据存储在**内存**中，程序关闭后数据会丢失，重启后恢复初始状态
2. 配送员接单后无法主动取消订单
3. 需求方在订单"待接单"状态时可取消订单，不产生任何费用
4. "配送中"的订单需配送员确认送达后才会变为"已完成"
5. 一个订单只能评价一次
